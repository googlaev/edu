# Лекция: Работа с параметрами командной строки в языке C

## Введение

Большинство программ, которые мы пишем, не работают в вакууме. Часто нам нужно передать данные в программу в момент её запуска, не используя ввод с клавиатуры во время работы. Например:
*   `ping google.com` (где `google.com` — параметр)
*   `gcc main.c -o program` (где `main.c`, `-o`, `program` — параметры)
*   `git commit -m "message"`

В языке C механизм передачи этих данных встроен прямо в точку входа программы — функцию `main`.

---

## 1. Теория: Signature функции main

Обычно вы видели `main` так:
```c
int main() { ... }
```
Но для работы с аргументами используется полная форма:

```c
int main(int argc, char *argv[]) { ... }
```
Или эквивалентно:
```c
int main(int argc, char **argv) { ... }
```

### Что такое `argc` и `argv`?

1.  **`argc` (Argument Count)** — целое число, количество аргументов, переданных программе.
2.  **`argv` (Argument Vector)** — массив указателей на строки (`char *`). Каждая строка — это один аргумент.

### Важное правило: Индексация
*   **`argv[0]`** — это всегда **имя самой программы** (путь к исполняемому файлу).
*   **`argv[1]`** — это первый реальный параметр, введенный пользователем.
*   **`argv[argc]`** — всегда равен `NULL` (конец массива).

**Визуализация памяти:**
Если мы запустим команду: `./app hello world`

| Индекс | Значение `argv[i]` | Описание |
| :--- | :--- | :--- |
| 0 | `"./app"` | Имя программы |
| 1 | `"hello"` | 1-й аргумент |
| 2 | `"world"` | 2-й аргумент |
| 3 | `NULL` | Конец массива |

`argc` в этом случае будет равен **3**.

## 2. Пример №1: Базовый вывод (Debug)

Давайте напишем программу, которая просто показывает, что она получила. Это полезно для отладки.

```c
#include <stdio.h>

int main(int argc, char *argv[]) {
    printf("--- Начало программы ---\n");
    printf("Количество аргументов (argc): %d\n", argc);

    // Проходим по всем аргументам от 0 до argc-1
    for (int i = 0; i < argc; i++) {
        printf("argv[%d] = %s\n", i, argv[i]);
    }

    printf("--- Конец программы ---\n");
    return 0;
}
```

**Как компилировать и запускать:**
```bash
gcc main.c -o app
./app foo bar 123
```

**Ожидаемый вывод:**
```text
--- Начало программы ---
Количество аргументов (argc): 4
argv[0] = ./app
argv[1] = foo
argv[2] = bar
argv[3] = 123
--- Конец программы ---
```

## 3. Пример №2: Ручная обработка флагов (Портативный способ)

В реальных задачах нам нужно не просто вывести аргументы, а найти конкретные ключи (флаги), например `-v` (версия) или `-n` (число).

Для сравнения строк в C используется функция `strcmp` из `<string.h>`. Она возвращает `0`, если строки равны.

```c
#include <stdio.h>
#include <string.h>
#include <stdlib.h> // Для atoi

int main(int argc, char *argv[]) {
    int verbose = 0;      // Флаг подробного вывода (по умолчанию выкл)
    int count = 1;        // Количество повторений (по умолчанию 1)
    
    // Начинаем с 1, потому что 0 - это имя программы
    for (int i = 1; i < argc; i++) {
        
        // Проверка на флаг -v
        if (strcmp(argv[i], "-v") == 0) {
            verbose = 1;
        }
        // Проверка на флаг -n
        else if (strcmp(argv[i], "-n") == 0) {
            // ВАЖНО: Проверка, есть ли следующий аргумент
            if (i + 1 < argc) {
                count = atoi(argv[i + 1]); // Преобразуем строку в число
                i++; // Пропускаем следующий аргумент, так как мы его "съели"
            } else {
                fprintf(stderr, "Ошибка: после -n должно быть число!\n");
                return 1;
            }
        }
        else {
            printf("Неизвестный аргумент: %s\n", argv[i]);
        }
    }

    if (verbose) {
        printf("[DEBUG] Режим отладки включен\n");
    }

    for (int i = 0; i < count; i++) {
        printf("Привет, мир! (iteration %d)\n", i + 1);
    }

    return 0;
}
```

**Примеры запуска:**
```bash
./app -v -n 3
# Выведет debug сообщение и 3 раза "Привет, мир!"

./app -n 10 -v
# Порядок флагов не важен!
```

## 4. Пример №3: Профессиональный подход (getopt)

Написать цикл с `strcmp` каждый раз неудобно. В стандарте POSIX (Linux, macOS, Unix) есть функция `getopt`, которая автоматически разбирает опции.

*Примечание: На Windows `getopt` отсутствует в стандартной библиотеке, но есть в MinGW или можно подключить совместимую библиотеку.*

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h> // Подключает getopt

int main(int argc, char *argv[]) {
    int opt;
    int number = 0;
    char *filename = NULL;
    int verbose = 0;

    // Строка "vn:f:" расшифровывается так:
    // v - флаг без значения (просто наличие)
    // n: - флаг требует значения (число)
    // f: - флаг требует значения (строка/файл)
    while ((opt = getopt(argc, argv, "vn:f:")) != -1) {
        switch (opt) {
            case 'v':
                verbose = 1;
                break;
            case 'n':
                number = atoi(optarg); // optarg - глобальная переменная со значением аргумента
                break;
            case 'f':
                filename = optarg;
                break;
            default:
                fprintf(stderr, "Использование: %s [-v] [-n число] [-f файл]\n", argv[0]);
                return 1;
        }
    }

    // Оставшиеся аргументы (не флаги) доступны через argv[optind]
    if (verbose) {
        printf("Режим отладки: ВКЛ\n");
    }
    if (filename) {
        printf("Работа с файлом: %s\n", filename);
    }
    printf("Число: %d\n", number);

    return 0;
}
```

**Преимущества `getopt`:**
1.  Автоматически объединяет флаги: `-v -n 5` можно написать как `-vn 5`.
2.  Автоматически проверяет наличие значений для флагов.
3.  Стандарт де-факто для консольных утилит в Linux.

## 5. Типичные ошибки и безопасность

При работе с `argv` легко получить ошибку сегментации (Segmentation Fault). Вот правила безопасности:

### 1. Проверка границ массива
Никогда не обращайтесь к `argv[i]`, не проверив, что `i < argc`.
```c
// ПЛОХО:
if (strcmp(argv[1], "-h") == 0) { ... } // А если argc == 1? Краш!

// ХОРОШО:
if (argc > 1 && strcmp(argv[1], "-h") == 0) { ... }
```

### 2. Преобразование строк в числа
Функция `atoi()` проста, но не сообщает об ошибках. Если пользователь введет `"abc"` вместо числа, `atoi` вернет `0`.
Лучше использовать `strtol()` (string to long):

```c
#include <stdlib.h>
#include <errno.h>

char *endptr;
long val = strtol(argv[i], &endptr, 10);

if (errno != 0 || *endptr != '\0') {
    printf("Ошибка: '%s' не является числом\n", argv[i]);
}
```

### 3. Имя программы
Помните, что `argv[0]` может быть разным в зависимости от того, как вызвали программу:
*   `./app` -> `argv[0]` = `"./app"`
*   `app` (если в PATH) -> `argv[0]` = `"app"`
*   `/usr/bin/app` -> `argv[0]` = `"/usr/bin/app"`
Не полагайтесь на точное значение `argv[0]` для логики программы.

## 6. Практическое задание

Попробуйте самостоятельно написать утилиту **калькулятор**, которая принимает аргументы в формате:
`./calc <число1> <операция> <число2>`

**Пример:**
```bash
./calc 10 + 5
# Вывод: 15

./calc 20 / 4
# Вывод: 5
```

**Подсказка:**
1.  Проверьте, что `argc == 4` (имя программы + 3 аргумента).
2.  Используйте `atoi` или `strtol` для перевода `argv[1]` и `argv[3]` в числа.
3.  Используйте `strcmp` для проверки `argv[2]` (там лежит `"+"`, `"-"`, `"*"` или `"/"`).

---

## Резюме

1.  Используйте сигнатуру `int main(int argc, char *argv[])`.
2.  `argv[0]` — имя программы, полезные данные начинаются с `argv[1]`.
3.  Для простых программ хватит цикла `for` и `strcmp`.
4.  Для сложных утилит на Linux используйте `getopt`.
5.  Всегда проверяйте `argc` перед доступом к `argv[i]`.
