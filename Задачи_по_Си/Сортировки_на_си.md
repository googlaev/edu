# Лекция: Алгоритмы сортировки на языке C

## Введение

Сортировка — это процесс упорядочивания элементов массива (или списка) в определенном порядке (обычно по возрастанию или убыванию). Это одна из фундаментальных задач в программировании.

**Зачем нужно сортировать?**
1.  **Поиск:** В отсортированном массиве поиск элемента (бинарный поиск) работает за $O(\log n)$, а в несортированном — за $O(n)$.
2.  **Анализ данных:** Упрощает поиск дубликатов, медианы, минимума/максимума.
3.  **Оптимизация:** Многие алгоритмы работают эффективнее на предварительно отсортированных данных.

В этой лекции мы рассмотрим основные алгоритмы сортировки, реализуем их на **чистом C** и проанализируем их эффективность.

---

## Основные метрики алгоритмов

При выборе алгоритма сортировки руководствуются тремя параметрами:

1.  **Временная сложность (Time Complexity):** Как быстро растет время выполнения при увеличении количества элементов $n$. Обозначается через $O$-нотацию (Big O).
    *   $O(n^2)$ — медленно (для больших данных).
    *   $O(n \log n)$ — быстро (стандарт для эффективных сортировок).
2.  **Потребление памяти (Space Complexity):** Сколько дополнительной памяти требуется.
    *   $O(1)$ — сортировка на месте (in-place).
    *   $O(n)$ — требуется дополнительный массив.
3.  **Устойчивость (Stability):** Сохраняет ли алгоритм относительный порядок элементов с одинаковыми ключами?
    *   *Устойчивая:* Да.
    *   *Неустойчивая:* Нет.

## 1. Пузырьковая сортировка (Bubble Sort)

**Идея:** Многократно проходим по массиву, сравнивая соседние элементы. Если они в неправильном порядке — меняем местами. "Тяжелые" элементы постепенно всплывают в конец, как пузырьки.

**Сложность:**
*   Время: $O(n^2)$ (в среднем и в худшем случае).
*   Память: $O(1)$.
*   Устойчивость: Да.

**Реализация на C:**

```c
#include <stdio.h>
#include <stdbool.h>

// Функция для обмена двух элементов
void swap(int* a, int* b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

void bubble_sort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool swapped = false; // Оптимизация: если обменов не было, массив уже отсортирован
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(&arr[j], &arr[j + 1]);
                swapped = true;
            }
        }
        if (!swapped) break;
    }
}
```

**Плюсы:** Простота реализации.
**Минусы:** Очень низкая производительность. На практике почти не используется.

## 2. Сортировка выбором (Selection Sort)

**Идея:** Находим минимальный элемент в неотсортированной части массива и меняем его местами с первым элементом этой части. Граница отсортированной части расширяется.

**Сложность:**
*   Время: $O(n^2)$.
*   Память: $O(1)$.
*   Устойчивость: Нет (при обмене может нарушиться порядок равных элементов).

**Реализация на C:**

```c
void selection_sort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int min_idx = i;
        // Поиск минимума в оставшейся части
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[min_idx]) {
                min_idx = j;
            }
        }
        // Обмен, если минимум не на своем месте
        if (min_idx != i) {
            swap(&arr[i], &arr[min_idx]);
        }
    }
}
```

**Плюсы:** Минимальное количество обменов (максимум $n-1$).
**Минусы:** Медленная, неустойчивая.

---

## 3. Сортировка вставками (Insertion Sort)

**Идея:** Берем элемент и вставляем его в правильную позицию внутри уже отсортированной левой части массива, сдвигая элементы вправо. Аналогично тому, как мы сортируем карты в руке.

**Сложность:**
*   Время: $O(n^2)$ (в худшем), $O(n)$ (если массив почти отсортирован).
*   Память: $O(1)$.
*   Устойчивость: Да.

**Реализация на C:**

```c
void insertion_sort(int arr[], int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;

        // Сдвигаем элементы, которые больше key, вправо
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j = j - 1;
        }
        arr[j + 1] = key;
    }
}
```

**Плюсы:** Очень эффективна для маленьких массивов ($n < 50$) или почти отсортированных данных.
**Минусы:** Медленная на случайных больших данных.

## 4. Быстрая сортировка (Quick Sort)

**Идея:** Алгоритм "Разделяй и властвуй".
1.  Выбираем **опорный элемент** (pivot).
2.  **Разбиваем** массив: элементы меньше pivot — влево, больше — вправо.
3.  Рекурсивно применяем сортировку к левой и правой частям.

**Сложность:**
*   Время: $O(n \log n)$ (в среднем), $O(n^2)$ (в худшем, если pivot неудачен).
*   Память: $O(\log n)$ (на стек рекурсии).
*   Устойчивость: Нет.

**Реализация на C:**

```c
// Функция разбиения (Partition)
int partition(int arr[], int low, int high) {
    int pivot = arr[high]; // Выбираем последний элемент как опорный
    int i = (low - 1); // Индекс меньшего элемента

    for (int j = low; j <= high - 1; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(&arr[i], &arr[j]);
        }
    }
    swap(&arr[i + 1], &arr[high]);
    return (i + 1);
}

void quick_sort(int arr[], int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);

        quick_sort(arr, low, pi - 1);
        quick_sort(arr, pi + 1, high);
    }
}
```

**Плюсы:** Одна из самых быстрых сортировок общего назначения на практике. Работает на месте.
**Минусы:** Неустойчивая. В худшем случае деградирует до $O(n^2)$ (лечится рандомизацией pivot).

## 5. Сортировка слиянием (Merge Sort)

**Идея:** Алгоритм "Разделяй и властвуй".
1.  Рекурсивно делим массив пополам, пока не останутся одиночные элементы.
2.  **Сливаем** отсортированные половинки в один отсортированный массив.

**Сложность:**
*   Время: $O(n \log n)$ (гарантированно).
*   Память: $O(n)$ (требуется временный массив).
*   Устойчивость: Да.

**Реализация на C:**

```c
#include <stdlib.h> // Для malloc

void merge(int arr[], int l, int m, int r) {
    int n1 = m - l + 1;
    int n2 = r - m;

    // Создаем временные массивы
    int* L = (int*)malloc(n1 * sizeof(int));
    int* R = (int*)malloc(n2 * sizeof(int));

    for (int i = 0; i < n1; i++)
        L[i] = arr[l + i];
    for (int j = 0; j < n2; j++)
        R[j] = arr[m + 1 + j];

    int i = 0, j = 0, k = l;

    // Слияние временных массивов обратно в arr
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) { // <= обеспечивает устойчивость
            arr[k] = L[i];
            i++;
        } else {
            arr[k] = R[j];
            j++;
        }
        k++;
    }

    // Копирование оставшихся элементов
    while (i < n1) {
        arr[k] = L[i];
        i++; k++;
    }
    while (j < n2) {
        arr[k] = R[j];
        j++; k++;
    }

    free(L);
    free(R);
}

void merge_sort(int arr[], int l, int r) {
    if (l < r) {
        int m = l + (r - l) / 2; // Избегаем переполнения при (l+r)/2

        merge_sort(arr, l, m);
        merge_sort(arr, m + 1, r);

        merge(arr, l, m, r);
    }
}
```

**Плюсы:** Стабильная, гарантированная скорость $O(n \log n)$. Идеальна для связных списков.
**Минусы:** Требует дополнительной памяти $O(n)$. Медленнее Quick Sort на массивах из-за аллокаций.

## 6. Стандартная библиотечная функция `qsort`

В языке C не нужно писать сортировку самому для продакшена. В `<stdlib.h>` есть функция `qsort`.

**Прототип:**
```c
void qsort(void *base, size_t nmemb, size_t size, int (*compar)(const void *, const void *));
```

**Пример использования:**

```c
#include <stdio.h>
#include <stdlib.h>

// Функция сравнения для qsort
// Должна возвращать: <0 если a<b, 0 если a==b, >0 если a>b
int compare_ints(const void* a, const void* b) {
    int arg1 = *(const int*)a;
    int arg2 = *(const int*)b;

    if (arg1 < arg2) return -1;
    if (arg1 > arg2) return 1;
    return 0;
    // Или короче: return (*(int*)a - *(int*)b); 
    // Но осторожно с переполнением при вычитании больших чисел!
}

int main() {
    int arr[] = {64, 34, 25, 12, 22, 11, 90};
    int n = sizeof(arr) / sizeof(arr[0]);

    qsort(arr, n, sizeof(int), compare_ints);

    printf("Отсортированный массив: \n");
    for (int i = 0; i < n; i++)
        printf("%d ", arr[i]);
    printf("\n");

    return 0;
}
```

**Важно:** `qsort` обычно реализована через Quick Sort или Hybrid Sort (например, Timsort в glibc), поэтому она оптимизирована лучше самописных вариантов.

## Сравнительная таблица

| Алгоритм | Среднее время | Худшее время | Память | Устойчивость | Когда использовать |
| :--- | :---: | :---: | :---: | :---: | :--- |
| **Bubble** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | Да | Только для обучения |
| **Selection** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | Нет | Когда важна минимизация обменов |
| **Insertion** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | Да | Малые массивы ($n < 50$), почти отсортированные |
| **Quick** | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | Нет | Общий случай, работа с памятью ограничена |
| **Merge** | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | Да | Когда нужна стабильность или работа со списками |
| **qsort** | $O(n \log n)$ | Зависит | - | Нет | **В продакшене** |

## Заключение и рекомендации

1.  **Для учебы:** Реализуйте Bubble, Insertion и Quick Sort вручную, чтобы понять принципы работы циклов, рекурсии и указателей.
2.  **Для маленьких массивов:** Insertion Sort часто выигрывает у сложных алгоритмов из-за отсутствия накладных расходов.
3.  **Для реальных задач:** Всегда используйте стандартную `qsort` (в C) или `std::sort` (в C++). Самописные сортировки сложнее отладить и оптимизировать.
4.  **Сортировка структур:** При сортировке массивов структур `qsort` незаменима. Вы просто меняете функцию сравнения.

### Пример сортировки структур

```c
typedef struct {
    char name[50];
    int age;
} Person;

int compare_people(const void* a, const void* b) {
    Person* p1 = (Person*)a;
    Person* p2 = (Person*)b;
    // Сортировка по возрасту
    return p1->age - p2->age;
}
```
